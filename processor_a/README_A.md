# ESP32S3_N16R8_rs485_2 项目说明

这是一个基于 **ESP32-S3** 的多功能嵌入式物联网综合工程，集成了传感器采集、电机控制、边缘 AI 推理和蓝牙通信，适用于工业自动化、机器人控制及智能设备原型开发等场景。

---

## 硬件平台

| 项目 | 说明 |
|------|------|
| **主控** | ESP32-S3 N16R8（16MB Flash + 8MB PSRAM） |
| **开发框架** | PlatformIO + Arduino |
| **串口波特率** | 115200 |
| **分区表** | 16MB Flash 自定义分区 |

---

## 功能模块一览

工程包含 **8 大功能模块**，每个均可通过宏开关独立启用/禁用：

| 模块 | 宏开关 | 默认状态 | 说明 |
|------|--------|:--------:|------|
| 按键检测 | `__Key__` | ❌ 关闭 | 物理按键输入 |
| 麦克风采集 | `__MIC__` | ✅ 启用 | GPIO 7 音频输入 |
| MPU6050 姿态 | `__MPU__` | ✅ 启用 | I2C（SDA=4, SCL=5），输出 Yaw/Pitch/Roll、加速度、角速度 |
| 边缘 AI 推理 | `__EdgeInf__` | ❌ 关闭 | 基于 TensorFlow Lite Micro，运行在 Core 1 |
| INA219 监测 | `__INA__` | ✅ 启用 | I2C（SDA=8, SCL=9），监测电压/电流/功率 |
| DHT11 温湿度 | `__DHT11__` | ✅ 启用 | GPIO 19 读取温湿度 |
| 步进电机控制 | `__Motor__` | ✅ 启用 | RS485/UART 闭环控制（SMD 协议） |
| BLE 透传 | `__BLE__` | ✅ 启用 | 蓝牙设备名 `ESP32S3_Host_QS`，多连接支持 |

## 通信架构

### 三路 UART

| 接口 | TX | RX | 用途 |
|------|:--:|:--:|------|
| **MCU1_UART** | 16 | 15 | 接收远程指令（如 `wf`、`Go+`、`buzzer` 等文本命令） |
| **MCU2_UART** | 41 | 42 | 步进电机驱动器通信（SMD 协议，帧头 `0xC5`，帧尾 `0x5C`） |
| **MCU3_UART** | 13 | 14 | 预留扩展串口 |

### BLE 蓝牙

- UUID: `0xFFE0`
- 支持最多 **4 个设备**同时连接
- 配套 [ble_client.py](ble_client.py) 可实现 Windows PC 与 ESP32 的无线通信

---

## 电机控制子系统

基于 **PD42S1 闭环步进电机驱动器**（正点原子 SMD 协议），支持以下控制命令：

| 命令 | 说明 |
|------|------|
| `f +0.1` / `b -0.1` | 前/后移动指定圈数 |
| `s` / `start` | 启动电机 |
| `p` / `stop` | 停止电机 |
| `z` / `zero` | 回零操作 |
| `wf 速度 加速度 圈数` | 往复运动模式 |
| `QzStop` / `QzUnstop` | 急停制动 |

### 实时监控参数

- 相电流 / 转速 / 位置
- 使能状态 / 到位状态 / 在线状态
- 支持设置软件限位（高/低限位）

---

## 边缘 AI 推理（Edge Inference）

- 基于 **TensorFlow Lite Micro**（ESP32 官方移植版）
- 模型数据存储在 `model_data[]` 数组（[model_param.h](include/model_param.h)）
- 推理配置：
  - **7 分类**，**15 维特征**，**10 窗口滑动**
  - Tensor Arena：**40 KB**
- 支持模拟数据模式（`EDGE_ENABLE_SIMULATION=1`）和外部输入模式
- 推理任务独立运行在 **Core 1**

## 软件架构特点

```
┌─────────────────────────────────────────────────────────┐
│  ESP32-S3 双核                                           │
│  ┌───────────────┐    ┌───────────────────────────────┐ │
│  │   Core 0      │    │   Core 1                      │ │
│  │  (协议栈/系统) │    │   Edge_Inference Task         │ │
│  └───────────────┘    └───────────────────────────────┘ │
│                                                         │
│  各传感器独立 FreeRTOS 任务：                            │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐        │
│  │ MPU  │ │ INA  │ │ DHT  │ │ MIC  │ │Motor │        │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘        │
└─────────────────────────────────────────────────────────┘
```

- **模块化设计**：每个功能模块拥有独立的 `.h/.cpp` 文件，放在 `include/` 目录
- **编译时裁剪**：通过预处理器宏控制各模块启用/禁用
- **调试友好**：每个模块自带独立调试打印开关（如 `__MPUDebug__`、`__INADebug__`）
- **主循环（`loop()`）**：
  - 检查电机往复模式并执行
  - 处理远程 UART 命令（MCU1）
  - 运行边缘推理
  - 每 100ms 打印各模块调试数据

## 快速开始

1. 安装 [PlatformIO](https://platformio.org/)（VS Code 插件或 CLI）
2. 克隆本仓库
3. 根据需求编辑 `include/config.h` 中的宏开关
4. 编译并烧录：
   ```bash
   pio run -t upload
   ```
5. 打开串口监视器查看调试输出：
   ```bash
   pio device monitor -b 115200
   ```

## 项目总结

这是一个 **功能丰富、模块化程度高** 的 ESP32-S3 综合工程，融合了：

- ✅ 多传感器数据采集（MPU6050、INA219、DHT11、麦克风）
- ✅ 闭环步进电机控制（RS485/SMD 协议）
- ✅ 边缘 AI 推理（TensorFlow Lite Micro）
- ✅ 蓝牙 BLE 透传（多连接）
- ✅ 多路 UART 指令交互
- ✅ 双核 FreeRTOS 任务调度

适用于需要 **感知 + 控制 + 智能推理 + 无线通信** 的复杂物联网设备开发。
